Why you should hire a farmer

Debugging code is a lot like debugging plants - specifically, removing hornworms from tomato plants. First you have to test the code. This is like walking into a field of tomatoes, hoping that there is no damage, but knowing that Murphy's law exists. Then you have to find and correct the error: carefully check every leaf and line of code until you catch the suspect red-handed. In both fields, you never know if you have caught the last bug. I have developed an extremely detail oriented mindset because bugs tend to multiply, and I do not like creating larger problems for myself. Beyond searching for bugs, I am used to working long hours in physically and mentally uncomfortable conditions. Farming means being outside in all types of weather and solving an endless array of problems. While I anticipate your office will be air conditioned, being able to maintain focus and a good attitude when facing challenging and inconvenient situations is a critical skill to form desirable solutions. I am determined, scrappy, have a solid foundation in software development, and am excited to jump into your role.

React Flask Language Learning Web App

Click here to see the language learning web app

I created an app that allows users to select a Gutenberg Library book from a bookshelf, and then read it in any language. Users can customize light mode vs dark mode, difficulty of the text, font, language, and the page number which they are reading. Additionally, users can click on words they are unfamiliar with which translates the word to english and adds it to a list of "mistakes" which they can review with a built in flash cards deck.

    Easy to turn pages

    In order to make it easy for users to find the page they are looking for, I used a number input for users to type in a specific page.

    To make is simple to turn a page, I created an image of the bottom of a page turning that is visible when a user mouses over it. I also added a small blank footer at the bottom of the page, so that this image does not interfere with the user clicking on any words. When the user clicks on the image in the bottom left corner, the page number decrements by one, and when the user clicks on the image in the bottom right corner, the page number increments by one.
